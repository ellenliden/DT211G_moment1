Min arbetsprocess

Inledning

Min arbetsprocess är strukturerad för att effektivisera utvecklingen och publiceringen av webbplatsen genom att använda Git, Node.js och automatiseringsverktyg. Genom att arbeta med branches, automatisera uppgifter och integrera en CI/CD-lösning (Continuous Integration/Continuous Deployment) säkerställer jag en smidig och pålitlig arbetsflöde.

Hur tillgänglig är webbplatsen?

Jag använder Git och GitHub för att hantera versionshistorik och samarbeta effektivt. För att säkerställa en tydlig struktur i projektet följer jag dessa principer:

  • Branch-struktur: Jag arbetar med två huvudbrancher: development för aktiv utveckling och main för publicering.
  • Regelbundna commits: Varje ändring committas med tydliga och beskrivande meddelanden.
  • Exkludera filer med .gitignore: Jag exkluderar filer som inte ska versionshanteras, exempelvis node_modules/.
  • Pull requests & merges: Jag använder pull requests för att granska och testa kod innan den slås samman med main. När jag vill föra över kod från development till main skapar jag en pull request. När den godkänns och mergas blir koden en del av den slutgiltiga versionen.

Automatisering med Node.js

För att effektivisera arbetsflödet använder jag Node.js tillsammans med automatiseringsverktyg. Dessa verktyg används för att hantera byggprocessen, optimera resurser och säkerställa att webbplatsen fungerar smidigt.

Automatiserade uppgifter

Jag har konfigurerat följande automatiserade uppgifter:

  • Minifiering av CSS och HTML med Parcel för att reducera filstorlek.
  • Transpilering och minifiering av JavaScript med Parcel.
  • Bildoptimering och konvertering till WebP och AVIF med Parcel.
  • Kopiering av optimerade filer till en publiceringsmapp (dist/).

Dessa uppgifter körs automatiskt när jag bygger projektet med "npm run build" i terminalen. För att starta en utvecklingsserver lokalt använder jag "npm run start". Detta startar en server som uppdaterar sidan automatiskt när jag gör ändringar i koden.

Publicering

Jag använder Netlify för att automatisera publiceringen av webbplatsen. Tjänsten är kopplad till mitt GitHub-repository och övervakar main-branchen, vilket innebär att varje ändring som pushas automatiskt byggs och publiceras. När jag har gjort uppdateringar i dev-branchen och är nöjd med resultatet, skapar jag en pull request till main. När den mergas, triggas en ny publicering, och webbplatsen uppdateras med de senaste ändringarna.

Projektstruktur

Projektet är organiserat för att vara lättöverskådligt och följa en tydlig struktur.

  • src/ – Innehåller all källkod och resurser som används i projektet.
    • images/ – mapp som lagrar alla bilder och andra medier.
    • styles.css – stilfilen för all styling
    • app.js – JavaScript-filen för all javascript på hemsidan
    • index.html, gallery.html och process.html – dvs alla sidor med innehåll för webbplatsen.
  • dist/ – Den mapp där den färdigbyggda webbplatsen hamnar efter att projektet har byggts med Parcel.
  • node_modules/ – Innehåller alla beroenden som installeras via npm.
  • .parcel-cache/ – Cachefiler som används av Parcel för att snabba upp byggprocessen.
  • .gitignore – Lista över filer och mappar som inte ska inkluderas i versionshanteringen med Git.
  • package.json – Beskriver projektets beroenden och inställningar.
  • package-lock.json – Sparar exakta versioner av installerade paket för att säkerställa konsekventa installationer.
  • README.md – Innehåller dokumentation om projektet. Denna struktur hjälper till att hålla projektet organiserat, vilket gör det enklare att hantera och vidareutveckla.
